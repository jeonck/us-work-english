# 업무 이메일

## 이메일 구조

```
Subject: [명확하고 구체적인 제목]

Hi [Name], / Hello [Name],   ← 인사

[핵심 내용 - 1~2문장]

[세부 내용 또는 요청]

[마무리 문장]

Best,                         ← 서명
[Your Name]
```

---

## 상황별 표현

### 요청할 때

```
Could you please review the attached document by Friday?
Would it be possible to reschedule our meeting to Thursday?
I wanted to check if you've had a chance to look at my previous email.
```

### 정보 전달할 때

```
Just a heads-up — the meeting has been moved to 3 PM.
Wanted to loop you in on the latest update.
FYI, the client approved the proposal this morning.
```

### 확인/감사할 때

```
Thanks for the quick response!
I really appreciate your help with this.
Got it, thanks!
```

### 마감 연장 요청할 때

```
I wanted to reach out about the deadline for [project].
Due to [reason], I may need a few more days.
Would it be okay if I submitted it by [new date] instead?
```

## 제목(Subject) 잘 쓰는 법

| 나쁜 예 | 좋은 예 |
|---------|---------|
| Question | Question about Q3 Budget Report |
| Update | Project X — Status Update (Week 3) |
| Meeting | [Action Required] Meeting Reschedule Request |
| Hi | Following up on our conversation |

---

## 이메일 톤 조절

| 상황 | 표현 |
|------|------|
| 공식적 (임원, 외부) | "I would like to request..." / "Please let me know if..." |
| 일반 업무 | "Could you...?" / "I wanted to..." |
| 친한 동료 | "Hey, any chance you can...?" / "Quick question —" |

:::tip Reply vs Reply All
회사 전체에 보낼 필요가 없다면 **Reply**만 사용하세요.
"Reply All"은 필요한 사람만 포함될 때 씁니다.
:::
