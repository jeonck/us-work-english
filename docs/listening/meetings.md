# 회의에서 놓치지 않고 따라가기

## 회의 듣기의 어려움

- 여러 사람이 번갈아 말함
- 빠른 속도로 주제가 전환됨
- 전문 용어와 이디엄이 섞임
- 웃음·농담 타이밍을 맞춰야 함

---

## 핵심 전략

### 1. 의제(Agenda) 미리 파악하기
회의 전 안건을 알면 모르는 단어가 나와도 맥락으로 파악 가능합니다.

```
회의 초대 이메일에서 확인하거나:
"Could you share the agenda beforehand?"
```

### 2. 키워드 중심으로 듣기
모든 단어를 다 들으려 하지 말고 **명사·동사·숫자**를 집중해서 듣습니다.

```
"We need to ship the feature by Q3 or we'll lose the contract."
           ^^^^^^^^^^^^^^^^^^^^  ^^^       ^^^^^^^^^^^^^^^^^^^
           핵심 키워드             기한       결과
```

### 3. 노트 테이킹
들으면서 키워드와 숫자만 메모합니다.

```
좋은 노트 예시:
- deadline: Q3
- owner: Sarah
- blocker: API integration
- next: sync Thursday

나쁜 노트 예시:
(전체 문장을 그대로 받아 적으려다 놓침)
```

---

## 못 들었을 때 쓰는 표현

| 상황 | 표현 |
|------|------|
| 전체를 못 들었을 때 | "Sorry, I didn't catch that. Could you repeat?" |
| 마지막 부분만 놓쳤을 때 | "I got the first part, but could you repeat the last bit?" |
| 단어가 이해 안 될 때 | "What do you mean by [word]?" |
| 확인할 때 | "Just to clarify — are you saying [요약]?" |
| 스펠링 확인 | "Could you spell that for me?" |

:::caution 자주 하는 실수
"What?" 또는 "Huh?" 는 너무 짧고 무례하게 들릴 수 있습니다.
`"Sorry, could you say that again?"` 으로 대신하세요.
:::

## 회의 후 확인하기

완전히 이해 못 해도 괜찮습니다. 회의 후 개인적으로 확인하세요.

```
Slack/이메일로:
"Hey [Name], quick question from the meeting —
when you mentioned [topic], did you mean [해석 A] or [해석 B]?"

"Just wanted to confirm my action item:
I'm responsible for [task] by [date], right?"
```
