# 미국 이력서 작성법

## 핵심 원칙

미국 이력서는 **한국 이력서와 완전히 다릅니다.**

- 사진 없음, 나이 없음, 성별 없음
- 1페이지 (경력 10년 이하)
- 성과를 숫자로 표현

---

## 필수 섹션

### 1. Summary (요약)

```
Results-driven software engineer with 5+ years of experience building
scalable web applications. Passionate about clean code and team collaboration.
```

### 2. Experience (경력)

**작성 공식: 동사 + 무엇을 + 결과(숫자)**

```
✅ Reduced API response time by 40% by optimizing database queries
✅ Led a team of 4 engineers to deliver the payment module 2 weeks ahead of schedule
✅ Increased user retention by 25% through A/B testing and UX improvements

❌ Responsible for API development (결과가 없음)
❌ Worked on database (너무 모호)
```

### 3. Skills

```
Languages: Python, JavaScript, TypeScript
Frameworks: React, Node.js, Django
Tools: Git, Docker, AWS, Jira
```

---

## 자주 쓰는 동사 (Action Verbs)

| 카테고리 | 동사 |
|----------|------|
| 개발 | Built, Developed, Implemented, Designed, Architected |
| 개선 | Optimized, Improved, Streamlined, Reduced, Increased |
| 리더십 | Led, Managed, Mentored, Coordinated, Spearheaded |
| 분석 | Analyzed, Evaluated, Researched, Identified |

---

## 체크리스트

- [ ] 동사는 과거형 (Built, Led, Designed)
- [ ] 숫자가 최소 3개 이상 포함
- [ ] 맞춤법 검사 완료 (Grammarly 활용)
- [ ] 지원하는 JD의 키워드가 이력서에 포함
